# Options Analysis for the Rivet Risk CMS

## 1. Executive Summary
* **Decision Context:** The Rivet website (https://rivetrisk.com.au/) is a mess of AI generated code and images. It does not present a clean modern interface to the visitor. The site is custom code with no content capability.
* **Goal:** Rivet Risk Services marketing is driven by a seamless end-to-end capability that supports the full sales funnel from Awareness to Customer Retention.
* **Recommendation Summary:** Which option is the top choice and why?

I got AI to produce an in depth options analysis. The best bet for me is to code using the Astro framework on a Sveltia headless CMS. If I combine this with AI generated elements and refactor heavily. I can max my productivity. The principles of development are:
- Reuse what works
- Abstract components as far as possible
- Refactor continuously
- Use best of breed that maximises cost to realisable benefit
